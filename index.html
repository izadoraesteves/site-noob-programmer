Git para iniciantes

O que é Git?

Se você é igual eu e caiu de paraquedas dentro do mundo de Git, saiba que a imagem transparecida de bicho papão é uma grande farsa! A realidade é que o git e todas as suas ferramentas merecem ser estudadas com atenção e dedicação, pois caso o contrário, você pode se perder completamente!

O que é git de fato?

O git é um sistema de controle de versões distribuído, usado principalmente no desenvolvimento de software. O Git foi inicialmente projetado e desenvolvido por Linus Torvalds para o desenvolvimento do sistema operacional Linux!

Diferente de outros sistemas de controle em versão centralizada, como SVN e CVS, o Git é distribuído: todo desenvolvedor tem o histórico completo de seu repositório de códigos no local.

Você deve estar se perguntando "Mas quais são as vantagens?", bem vamos te explicar:

* Controle de histórico;
* É seguro;
* Trabalho em equipe dividido;
* É possível ramificar os projetos;

Legal, né?! Continua lendo, pois vamos mostrar alguns comandos essenciais para você começar a usar o seu Git!

Git Init

O `git init` é um comando utilizado para iniciar um novo repositório ou um que já tenha existência.
Existe também opções que podem seguir junto com o `git init`, que são eles:

`git init [-q | --quiet] [--bare] [--template = ] [--separate-git-dir ] [--object-format = ][-b | --initial-branch = ] [--shared [= ]] [diretório]`

Git Clone

O `git clone` cria uma cópia de trabalho em um repositório local executando o comando `git clone /caminho/para/o/repositório`
mas quando for usar um repositório remoto, seu comando será `git clone usuário@servidor: /caminho/para/o/repositório`

Git add

Usado para adicionar conteúdo do arquivo ao índice.
Existe também opções que podem seguir junto com o `git add`, que são:

` git add [--verbose | -v] [--dry-run | -n] [--force | -f] [--interactive | -i] [--patch | -p] [--edit | -e] [- [não-] todos | - [no-] ignorar-remoção | [--update | -você]] [--intent-to-add | -N] [--refresh] [--ignore-errors] [--ignore-missing] [--renormalize] [--chmod = (+ | -) x] [--pathspec-from-file = [--pathspec-file-nul]] [-] […] `

Git Commit

É usado para gravar as alterações feito dentro do repositório local.
Existe também opções que podem seguir junto com o `git commit`, que são eles:

` git commit [-a | --interactive | --patch] [-s] [-v] [-u] [--amend] [--dry-run] [(-c | -C | --fixup | --squash) ] [-F | -m ] [--reset-author] [--allow-empty] [--allow-empty-message] [--no-verify] [-e] [--author=] [--date=] [--cleanup=] [--[no-]status] [-i | -o] [--pathspec-from-file= [--pathspec-file-nul]] [-S[]] [--] […​]`

Git Push

Esse comando atualiza o seu repositório remoto com base nas atualizações do seu repositório local, ou seja, ele manda todas as configurações e comandos que você executou dentro da máquina.
Existe também opções que podem seguir junto com o `git push`, que são:

` git push [--all | --mirror | --tags] [--follow-tags] [--atomic] [-n | --dry-run] [--receive-pack=] [--repo=] [-f | --force] [-d | --delete] [--prune] [-v | --verbose] [-u | --set-upstream] [-o | --push-option=] [--[no-]signed|--signed=(true|false|if-asked)] [--force-with-lease[=[:]]] [--no-verify] [ […​]] `

Git Status

Esse comando mostra a condição de como se encontra a sua área de trabalho, exibindo assim, os caminhos e as diferenças.
Nesse comando, também existe opções que podem se seguir junto com o `git status`, são eles:

`git status […​] [--] […​]`

Git Merge e o Git Branch

Antes de explicar o que é um Merge, é necessário entender o que é um Branch, ou melhor, entender o que é uma ramificação.

Quando se inicia um novo projeto no git, você tem um caminho que já está criado e que sempre consideramos ele como principal, o Master. Ele será a sua última versão do projeto.

Mas trabalhando em grupo (ou até mesmo sozinho), deve pensar na separação de setores, como: design, conteúdo, estrutura, entre outros. Assim, a sua árvore de trabalho terá uma estrutura mais profissional e organizada!

O merge aparece após essa criação de ramos, é esse comando que fará a junção com todos eles, apresentando o Master Final

Nesse comando, existe opções que podem se seguir junto com o `git merge`, são eles:

`git merge [-n] [--stat] [--no-commit] [--squash] [--[no-]edit] [--no-verify] [-s ] [-X ] [-S[]] [--[no-]allow-unrelated-histories] [--[no-]rerere-autoupdate] [-m ] [-F ] […​] git merge (--continue | --abort | --quit)`

Git Pull

Com esse comando você consegue integrar as suas alterações em um outro repositório ou em outro ramo local. Basta utilizar esse comando:

`git pull [] [ […​]]`
